The System

The 3-Layer Model

L1

Inputs

Raw business data from many sources — scattered across tools by default. The portal gives them a home.

Compile · sync · structure
L2

Knowledge

Markdown is the source of truth. Sync scripts compile it into portal data. AI maintains it — you direct.

Surface · decide · ship
L3

Insights & decisions

Not a dashboard you stare at. A system you operate through.

The key insight: you already have layers 1 and 3. The missing piece is layer 2 — a structured knowledge layer that connects inputs to insights and decisions. The diagram is the same three-plane visual as the case-study page.
Layer 1

Inputs

The raw material. Everything your business already produces.

  • CRM notes & deal history
  • Financial reports & production-run data
  • Strategy & planning docs
  • Operations & SOPs
  • Marketing copy & brand guidelines
Layer 1 is what you already have. The problem is it lives everywhere and requires a search every time you need to act.
Layer 2

Knowledge

Markdown as source of truth. Compile, don't copy.

inputs/90-day-plan.md → sync-plan.js → portal data → planning.html
  • One file is the source. Everything else is compiled.
  • AI maintains the structure. You direct the decisions.
  • Git is your version history. Cursor is your editor.
Layer 2 is the novel part. Instead of copying data between tools, you maintain one markdown file and compile views from it. This is what makes the system compound.
Layer 3

Insights & decisions

Decision-grade views you open when you're about to act.

View 1

Dashboard

  • Revenue MTD
  • Active accounts
  • Top SKU
View 2

Pipeline

  • Accounts by status
  • Last contact
  • SKUs carried
View 3

Finance

  • Revenue by run
  • Pricing emulator
  • Margin by SKU
Layer 3 is just a website. Static HTML. But because it's compiled from your markdown source, it's always current — and it's designed for the insights and decisions you actually need to make.

One token file, every page

Palette × mode stay orthogonal — change a token and the whole portal moves together.

Design Principles

Four rules that make it work

01

Markdown is the source of truth

Portable, diffable, AI-friendly.

02

Compile, don't copy

One edit propagates everywhere via sync scripts.

03

AI maintains; you direct

The model handles syncing; you own the decisions.

04

Own your infrastructure

No SaaS lock-in; everything in a git repo you control.

+ Every exploration adds up — sessions feed back into the knowledge base instead of evaporating in chat.

These aren't philosophical. They're the technical choices that make the system robust. If you violate any of them, the system breaks down within a few months.
