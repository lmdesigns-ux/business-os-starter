# Moxie project template — Nonprofit service mapping

**Template name in Moxie:** `DG — Service mapping (standard)`

**Product SKU:** DG-MAP-STD (fixed) · optional add-on DG-FACIL-DAY

**Fee type:** Fixed (or Per item if mixing add-on day)

**Naming when live:** `[Client] — Service mapping — [Year]`

---

## Tasks (in order)

| # | Task | Offset from start | Description |
|---|------|-------------------|-------------|
| 1 | Discovery & scoping call | Day 0 | Confirm goals, attendees, and which service to map first |
| 2 | Pre-workshop interviews (optional) | Day 1–7 | 2–3 stakeholder interviews if needed |
| 3 | Workshop 1 — Current state map | Day 7–14 | Facilitate as-is process / service map |
| 4 | Workshop 2 — Blueprint & gaps | Day 14–21 | Service blueprint; pain points and opportunities |
| 5 | Synthesis & draft deliverable | Day 21–28 | Draft blueprint + improvement backlog |
| 6 | Client review & revisions | Day 28–35 | One revision round |
| 7 | Final handoff & readout | Day 35–42 | Live readout; deliver PDF/Figjam/Miro exports |

## Sub-tasks (optional under task 5)

- Anonymize examples for portfolio
- Align map to grant or strategic plan language (if client requested)

## Portal / client visibility

- Toggle **client portal** on for tasks 6–7 only, or full project per client preference

## Agreement block

Insert in proposal:

- Scope: tasks 1–7
- Fixed fee: DG-MAP-STD rate from library
- Timeline: ~6 weeks from kickoff
- Client provides: staff availability, existing docs, system access for discovery
